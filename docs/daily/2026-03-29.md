# Session Summary — 2026-03-29

## Changes Made

### 1. AI Analyzer Tab — new feature
- **New route** `GET /polymarket/ai-analyzer` (`polymarket_ai_analyzer()`) added to `app/routes/polymarket.py`
- **New API endpoint** `POST /polymarket/ai-analyze` — calls Anthropic API server-side (key never sent to browser), uses `web_search` tool to pull live match news, returns JSON risk screen with verdict, confidence, stake recommendation, supporting factors, red flags
- **New template** `app/templates/polymarket_ai_analyzer.html` — full UI for the tab
- **`anthropic>=0.40.0`** added to `requirements.txt`
- **`STRATEGY_LABELS`** added to imports in `polymarket.py` routes (was missing, needed for nav dropdown)
- `"ai-analyzer"` added to `POLYMARKET_SECTIONS` list (powers top nav)
- AI Analyzer link added to **left sidebar** in `app/templates/base.html` (was missing — bug fixed this session)

### 2. Pick / Outcome display — Portfolio tab
- `format_resolution_result_detail()` added to `app/utils/polymarket.py`
  - Ties YES/NO resolution to the outcome side stored at alert time (e.g. "YES — Over hit", "NO — Chelsea lost")
  - Gracefully degrades to plain YES/NO for legacy rows without `outcome_label`
- `outcome_label`, `pick_display`, `result_detail_display` fields now populated on every resolved row in `get_polymarket_stats()` and every decision row in `get_recent_decisions()`
- **Portfolio template** (`polymarket.html`) updated:
  - Recent Decisions table: added **Pick** column
  - Resolved Bets table: added **Pick** column, improved Result badge to show `result_detail_display` instead of raw YES/NO, added **Sport** and **AI screen** columns (verdict badge + confidence)
  - `data-search` attribute on resolved rows now includes outcome_label and result_detail for client-side filtering

### 3. CSV Export improvement
- Export now includes `pick` (outcome_label) and `result_detail` columns alongside the existing `result` and `pnl_usd`
